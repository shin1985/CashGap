01

CashGap とは

CashGap は、年度ベースの資金繰り（キャッシュギャップ）管理ツールです。プロジェクト単位の売上計画を入力するだけで、PL（損益計算書）・BS（貸借対照表）・銀行残高予測・入金タイミング分析まで自動計算します。

特に「PL 上は黒字なのに、入金タイミングのズレで手元資金が不足する（黒字倒産リスク）」を先回りで可視化する点が最大の特長です。

図1: CashGap が解決する「PL黒字なのにキャッシュが足りない」問題
CashGap の3つの強み ① プロジェクト売上 → PL・入金予測まで自動連動
② 入金サイト（支払条件）を考慮したリアルなキャッシュフロー予測
③ Google Sheets に直接保存。サーバー不要でデータを完全にコントロール

初期セットアップ

CashGap を利用するには、Google Cloud の設定とローカル環境のセットアップが必要です。

01

Google Cloud プロジェクト作成

Google Cloud Console で新規プロジェクトを作成します。

02

API を有効化

Google Drive API / Google Sheets API / Google Picker API の3つを有効にします。

03

認証情報を作成

Web アプリケーション用の OAuth 2.0 クライアント ID と API キーを作成します。

04

環境変数を設定して起動

.env.local に3つの値を記入 → npm install → npm run dev

環境変数

VITE_GOOGLE_CLIENT_ID=あなたのOAuth_クライアントID
VITE_GOOGLE_API_KEY=あなたのAPIキー
VITE_GOOGLE_APP_ID=あなたのCloud_プロジェクト番号
⚠️ 環境変数が未設定の場合 Google Drive タブに警告バナーが表示されます。Google 連携を使わなくても、サンプルデータでの閲覧・編集は可能です。

08

🔁 入金サイト分析タブ

CashGap の核心機能です。PL 上の売上計上タイミングと、実際の入金タイミングのズレ（キャッシュギャップ）を可視化します。

図8: ギャップ分析テーブルとバーチャート
翌期繰越入金 入金サイトの影響で年度末までに入金されない売上がある場合、黄色い警告バナーでその金額が表示されます。年度末の資金計画を立てる際に重要な指標です。

黒字倒産リスクの発見

この分析の最も重要な目的は、PL上は利益が出ているのに手元資金が不足する「黒字倒産」リスクの早期発見です。

図8-2: 黒字倒産リスクの構造 — CashGap はこのギャップを事前に検出します

ギャップの読み方

ギャップ > 0（赤）

PL上は売上が計上されているが、まだ入金されていない状態。資金繰りが厳しくなる可能性があります。特に年度初めに大きくなりやすい傾向。

ギャップ < 0（緑）

前月以前の売上が今月入金された状態。入金超過でキャッシュは潤沢。前月のギャップが解消されたことを意味します。

よくある質問・トラブルシューティング

基本操作

セルの編集方法は？

数値セルをクリックすると入力モードになります。値を入力して Enter またはセル外クリックで確定します。確定するまで計算には反映されません。

サンプルデータに戻したい

Google Drive タブ内の「サンプルデータを投入」ボタンをクリックします。Google Sheets への保存は別途「保存」が必要です。

決算期首月を変更したら？

全タブの月ラベルが年度順に切り替わります（例: 4月始まり→3月決算、1月始まり→12月決算）。データの月別金額は Month1〜Month12 の順序が維持されるため、開始月に合わせて再入力が必要な場合があります。

データ管理

データはどこに保存される？

ユーザー自身の Google Drive 上の Google Sheets のみ。CashGap のサーバーには一切送信されません。アクセストークンもブラウザ内のみで使用されます。

Google Sheets 上で直接編集できる？

はい。CashGap のシートはシンプルなテーブル形式なので、Google Sheets 上で直接値を変更できます。変更後は CashGap で「再読込」を実行してください。

入金サイト関連

入金サイト「翌月末」とは？

売上計上月の翌月に入金されることを意味します。4月計上→5月入金。「翌々月末」なら4月計上→6月入金です。

翌期繰越入金とは？

入金サイトの影響で今期中に入金されない売上の合計です。例: 3月売上で翌月末サイト → 翌4月（翌期）に入金されるため、今期の残高には反映されません。

トラブルシューティング

Google 接続ができない .env.local に正しい値が設定されているか確認してください。必要な設定: VITE_GOOGLE_CLIENT_ID、VITE_GOOGLE_API_KEY、VITE_GOOGLE_APP_ID。OAuth 同意画面のスコープ（drive.file, spreadsheets）、リダイレクト URI（http://localhost:5173）、3つの API の有効化も確認してください。
保存ボタンが押せない スプレッドシートが未選択の状態です。Google Drive タブで「フォルダ選択」→「新規作成」または「既存を開く」を実行してください。
freee CSV が読み込めない freee の「月次推移」レポートの CSV であることを確認してください。Shift-JIS / UTF-8 どちらにも対応していますが、freee 以外のフォーマットには対応していません。sample/ フォルダのサンプル CSV と比較してフォーマットを確認できます。
入金額が期待と違う プロジェクト売上タブで各プロジェクトの入金サイト設定を確認してください。入金サイト分析タブでギャップの内訳を確認し、翌期繰越の金額も考慮してください。年度末に近い月の売上は翌期に繰り越されることがあります。
